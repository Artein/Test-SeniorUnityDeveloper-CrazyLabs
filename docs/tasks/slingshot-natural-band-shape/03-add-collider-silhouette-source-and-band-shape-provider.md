## What to build

Add the Unity-side Launch Target Silhouette source and the Band Shape provider that orchestrates Collider sampling, world-to-Pull-Plane mapping, pure solver invocation, and world-space output. The silhouette source should sample one explicitly assigned Collider at the band-height Pull Plane and write raw world samples into caller-owned buffers. The provider should own reusable scratch buffers sized from immutable config, map samples to solver `float2`, call the pure solver, and fill caller-owned world output buffers.

This slice should replace Unity-specific contact-provider internals without changing launch math.

## Acceptance criteria

- [x] `ILaunchTargetSilhouetteSource` receives a narrow geometry query with Pull Plane/basis/sample-count data, not a full gameplay Band Shape query.
- [x] `LaunchTargetSilhouetteQuery.PlaneOrigin` is geometric Pull Plane data, not `PullPoint` gameplay semantics.
- [x] The silhouette source supports one explicitly assigned Unity `Collider` of any Collider type and does not auto-discover child Colliders.
- [x] Raw samples are generated by radial sampling around the current Collider bounds center, using `Collider.ClosestPoint` from outside origins, then projecting samples to the band-height Pull Plane.
- [x] Too-small output buffers and invalid query values are setup/programmer errors; runtime Collider sampling failure returns false without throwing on the hot path.
- [x] The provider owns reusable world-sample and solver `float2` buffers and allocates/rebuilds them only during construction/config initialization.
- [x] Provider output writes into caller-owned `Vector3[]` buffers and returns point count instead of allocating/copying Band Shape arrays per frame.
- [x] Raw Collider sampling avoids unconditional `Physics.SyncTransforms`; add a sync only if a PlayMode boundary test proves same-frame Collider queries are stale after held-target movement.
- [x] Launch power, Pull Offset, launch direction, and Pull Point payloads remain unchanged by Band Shape provider behavior.

## Verification

- EditMode tests: provider maps solver output to caller-owned buffers; bad output buffers/query values fail fast; failed runtime solve returns false; shape point count is stable.
- PlayMode tests: same-frame held target movement is visible to Collider silhouette sampling; assigned Collider is used; child Colliders are not auto-selected; samples are constrained to Pull Plane/Band height; representative Collider types work through the base Collider contract where practical.
- Static checks: Rider reformat/problems on changed files; Unity compile via Unity AI Agent Connector before tests.
- Manual Unity smoke check: pull the Slingshot and inspect that the sampled Band Shape uses the assigned Launch Target Collider instead of scene-authored offsets.
- Package version/changelog: no package/changelog change.
